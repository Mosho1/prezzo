What is node.js?
node.js is platform built on Google's V8 JavaScript runtime for building network applications. It uses an asynchronous even-loop model, avoiding the long waiting periods when a server performs I/O operations.

How is it better?
Most web servers use OS threads or processes to handle requests. With every request, a new process/thread is spawned, incurring high time and memory cost due to context switching, execution stacks, and scheduling.

 


 I/O Latency in CPU cycles


 Request handling with threads






 node.js uses one process and an event loop instead




 



Examples

Hello HTTP
var http = require('http');

http.createServer(function (req, res) {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Hello World\n');
}).listen(1337, '127.0.0.1');

console.log('Server running at http://127.0.0.1:1337/');

Examples

File server
var http = require('http');
var fs = require('fs');

http.createServer(function (req, res) {
    fs.readFile('hello.txt', function(err, data) {
        if (err) {
            res.writeHead(500, err.message);
            res.end();
        } else {
            res.writeHead(200, {
                'Content-Type': 'text/plain'
            });
            res.end(data);
        }
    });
}).listen(1338);

console.log('Server running at http://127.0.0.1:1338/');
